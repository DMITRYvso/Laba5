Laba5/ - Корневая папка репозитория.
  .git/ - Служебная папка Git.
  .gitattributes Настройки Git.
  .gitignore Список игнорируемых файлов.
  Laba5.slnx - Файл решения для Visual Studio.
  Laba5/ - Основная папка проекта.
    Laba5.csproj - Файл проекта C#.
    Program.cs - Главный файл программы.
    (Другие .cs файлы) - Файлы с кодом классов.
    DataBase.txt - Основная база данных сотрудников.
      Сотрудники.txt - Файл с данными сотрудников.
      Экспорт.txt - Файл для экспортированных данных.
          bin/ - Папка для скомпилированных файлов.
          Debug/ или Release/ - В зависимости от конфигурации сборки.
      obj/ - Служебные объектные файлы компилятора
      Сначала выполните SQL-скрипт для создания таблицы в вашей базе данных через SQL Server Management Studio или аналогичный инструмент.
      Затем откройте проект Laba5 в Visual Studio, найдите файл конфигурации и обновите строку подключения к базе данных, указав правильное имя сервера и базы данных.
      Убедитесь, что выбрана правильная схема аутентификации (Windows или SQL Server).
      После этого восстановите зависимости NuGet через меню Visual Studio, соберите проект (Ctrl+Shift+B или dotnet build) и запустите его (F5 или dotnet run). 
      Проверьте, что программа запускается без ошибок подключения, и затем выполните тестовый запрос в SQL Server (например, SELECT * FROM Сотрудники;), чтобы убедиться в корректной работе с базой данных.
